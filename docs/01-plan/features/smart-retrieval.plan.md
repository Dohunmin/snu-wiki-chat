## 1. 현재 문제

```
쿼리 입력
  → 에이전트별 소스 전체 내용 전달
  → 소스 1개 = 3,000~4,000자
  → 쿼리 1회 = 최대 24,000자 (24K 토큰)
  → 비용: $0.10+/회
```

**핵심 낭비**: 소스 1개에서 쿼리와 관련된 문단은 1~2개인데, 전체 문서를 전달

---

## 2. 해결 방식: 청크 분할 + 관련 청크만 전달

### 2-1. 청크 분할 기준

```
소스 문서 (3,500자)
  ├── 청크1: ## 핵심 요약 ~ --- (400자)
  ├── 청크2: ## 1. 교육 분야 이슈 ~ ## 2. (600자)
  ├── 청크3: ## 2. 전략적 시사점 ~ ## 3. (700자)
  └── 청크4: ## 3. 전략과제 ~ 끝 (800자)
```

분할 기준:
- `##` 헤더 단위로 분할 (마크다운 구조 활용)
- 헤더 없는 경우 빈 줄 2개 단위
- 최소 100자 이상인 청크만 유효

### 2-2. 청크 점수 계산

```
쿼리 단어: ["평의원회", "이사회", "쟁점", "비교"]

청크1 점수: "평의원회" 2회, "이사회" 1회 = 3점
청크2 점수: "이슈" 1회 = 0점 (2자 미만 단어 제외)
청크3 점수: "쟁점" 1회, "비교" 1회 = 2점
청크4 점수: 0점
```

### 2-3. 전달 방식

```
기존: 소스 전체 (3,500자) × N개
개선: 관련 청크 (400~700자) × M개 (M은 제한 없음)
```

점수 0인 청크는 제외. 점수 있는 청크는 모두 전달 (소스 수 제한 없음).

### 2-4. 컨텍스트 보존

청크 전달 시 소속 소스 정보 포함:

```
## [평의원회] 17기-12차 — 선도연구진흥센터 부결
(원문 일부)
평의원회가 본부 발의 안건을 표결로 부결시킨 사례...
```

## 3. 예상 효과

| 지표 | 현재 | 개선 후 |
|---|---|---|
| 쿼리당 평균 토큰 | ~20,000 | ~5,000~8,000 |
| 비용/쿼리 | ~$0.08 | ~$0.02~0.03 |
| 소스 수 제한 | 에이전트당 2개 강제 | 제한 없음 |
| 관련 없는 내용 포함 | 많음 | 거의 없음 |

## 4. 구현 범위

**수정 파일**: `lib/agents/wiki-agent.ts` 1개만

**추가 함수**:
- `splitIntoChunks(content: string): string[]` — 마크다운 헤더 기준 분할
- `scoreChunk(chunk: string, queryWords: string[]): number` — 청크별 점수 계산

**제거**:
- `MAX_SOURCES = 2` 제한 완전 제거
- 기존 소스 단위 점수 계산 → 청크 단위로 교체

---

## 5. 엣지 케이스

| 상황 | 처리 |
|---|---|
| 관련 청크가 하나도 없음 | 각 소스의 첫 번째 청크(요약) fallback |
| 청크가 너무 많음 (20개 이상) | 상위 점수 15개 제한 |
| 헤더 없는 소스 | 빈 줄 2개 기준으로 분할 |
| 청크가 너무 짧음 (100자 미만) | 다음 청크와 합치기 |

---

## 6. 구현 순서

1. `splitIntoChunks()` 함수 구현 및 테스트
2. `scoreChunk()` 함수 구현
3. `getContext()` 내 소스→청크 처리 흐름 교체
4. `MAX_SOURCES` 제한 제거
5. 청크 출처 표시 형식 정리
